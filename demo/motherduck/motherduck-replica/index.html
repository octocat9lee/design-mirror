import duckdb

# Connect to DuckDB and attach MotherDuck
con = duckdb.connect()
con.execute("ATTACH 'md:'")

# Query to get 2025 revenue
query = """
SELECT
    EXTRACT(YEAR FROM o.order_date) as year,
    ROUND(SUM(od.unit_price * od.quantity * (1 - od.discount)), 2) as revenue
FROM orders o
JOIN order_details od ON CAST(o.order_id AS VARCHAR) = CAST(od.order_id AS VARCHAR)
WHERE EXTRACT(YEAR FROM o.order_date) = 2025
GROUP BY EXTRACT(YEAR FROM o.order_date)
"""

# Execute and print results
result = con.execute(query).fetchall()

HOW WE SCALE

DUCKLING SIZES

MotherDuck's per-user tenancy model gives each user an isolated duckling (DuckDB instance) in one of five sizes to enable individual, user-level configuration.

PULSE

Our smallest instance, perfect for ad-hoc analytics tasks

STANDARD

Built to handle common data warehouse workloads

JUMBO

For larger data warehouse workloads with many transformations

MEGA

An extremely large instance for complex transformations done quickly

GIGA

Largest instances enable the toughest transformations to run faster

PER-USER TENANCY AND VERTICAL SCALING

MotherDuck employs a per-user tenancy and vertical scaling strategy. Users connect to their own MotherDuck Ducklings (DuckDB instances), which are sized (pulse, standard, jumbo, mega, giga) to meet their specific needs.
